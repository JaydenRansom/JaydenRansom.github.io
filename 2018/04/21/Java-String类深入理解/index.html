String 类的三大特性

在 Java 开发中，String 类并不陌生。作为日常开发的最常用工具类之一，Java 在其的实现上，也进行了很多优化。
String 类底层是一个char数组，在类定义中声明如下：

value 数组便是实际的字符载体。
相比于其他类，String 类有三个基本特点：

  • 不变性：String 对象一旦生成，则不能对它进行改变；这个特点使得其在被多线程共享时，可以省略同步和锁等待的过程，提高性能
  • 针对常量池优化：拥有相同值的字符串引用常量池中的同一个拷贝，对于频繁出现的字符串，能够节省内存空间
  • 类的 final 定义：由于 String 被约定成 immutable （不可变）的，所以声明成 final 可以防止开发者继承 String，实现 mutable （可变）的子类

以上就是 String 类的三大特性，这三大特性也就成了 String 类的设计与实现基础。

String 对象的创建

一个 String 对象的创建过程如下：

  1. Java Runtime 在创建一个 String 对象时，会在 String 常量池中查找是否存在值相同的字符串对象。如果存在，则根据下面 2 或 3 过程进行执行；若不存在，则会在池中创建一个字符串
  2. 针对 使用 new 关键字 或 包含变量的表达式 创建 String对象：在 Java Heap[1] 中创建一个 String 对象
  3. 针对非使用 new 关键字创建 String对象：执行完 1 过程后，无后续 Java Heap 区的创建过程

String.intern() 方法的作用是保证每个通过此方法实例的字符串都是常量池中的引用，其具体的优点有两条：

  1. 由于字符串常量池中的字符串都是唯一的，所以全部使用 intern() 方法获取字符串会节省内存
  2. 通过 intern() 方法获取到的字符串可以使用 == 替代 equals() 进行比较，速度较快
    同时，由于 JVM 运行时内存分区的问题，使用 intern() 方法也会出现一些问题。
    了解过 JVM 运行时结构的都应该知道，
  • JDK 1.6 及以下：字符串常量池位于 Java Heap 的 PermGen(永久代) 区域，该区域是整个 JVM 实例共享的区域，如果使用 intern() 过多，会出现 OutOfMemoryError。
